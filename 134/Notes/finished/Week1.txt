"Redundancy" : ∙ Duplication, Waste Storage, Inconsitent
	-> When using file systems, redundancy is commonplace. When two groups need to access the same data, they must have two different file bases.
	-> This creates redundancy and when the two groups have to merge the files, some may have added different fields and have conflicts.
	-> When the two groups update the filesystem, they may have done it differently, and this may also cause inconsistent conflicts
